# Tarea 3.4 — CI con GitHub Actions + Dependabot

> Fase 3 — Tests y CI · 🟡 Ítem 16 del plan · OWASP A06/A08 · Depende de: 3.2, 3.3 · Reglas: [README.md](README.md)

## Contexto

Sin CI, nada garantiza que lo que se entrega compila y pasa los tests. Además OWASP A06 exige vigilancia continua de dependencias vulnerables: `pnpm audit` en el pipeline + Dependabot.

## Archivos

- `.github/workflows/ci.yml` (nuevo)
- `.github/dependabot.yml` (nuevo)

## Pasos

1. Workflow `ci.yml` disparado en `push` y `pull_request`:
   - **Job `backend`**: checkout → pnpm (con cache) → `pnpm install --frozen-lockfile` → `pnpm lint` → `pnpm build` → `pnpm test` → `pnpm audit --audit-level=high` (falla el pipeline si hay vulnerabilidades altas/críticas).
   - **Job `e2e`**: service container `postgres:15-alpine` (usuario/clave/db de test vía env del workflow) → migraciones → `pnpm test:e2e`. Las variables de entorno del job replican `.env.example` con valores de test (los secretos de test no son sensibles, pueden ir en el yaml).
2. `dependabot.yml`: ecosistema `npm`, directorio `/`, intervalo semanal, límite de PRs abiertas razonable (5).
3. **Frontend** (repo separado `e:\proyectos\control-produccion`): crear allí su propio `.github/workflows/ci.yml` (install → build) y `dependabot.yml`. Si los repos se unifican algún día, fusionar workflows.

## Criterio de aceptación

- [x] Pipeline verde en GitHub sobre la rama de trabajo (ambos jobs).
- [x] Una vulnerabilidad alta simulada (o real) hace fallar el pipeline.
- [x] Dependabot activo en ambos repos.
- [x] Commit: `ci: lint, build, tests, audit and dependabot`.
