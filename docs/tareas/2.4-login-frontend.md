# Tarea 2.4 — Login en el frontend

> Fase 2 — Seguridad perimetral · 🔴 Cierra el ítem 1 del plan · Depende de: 2.3 · Reglas: [README.md](README.md)
> Proyecto: **frontend** — `e:\proyectos\control-produccion`

## Contexto

Con los guards activos (tarea 2.2), el frontend necesita autenticarse: pantalla de login, token en cada request y manejo de sesión expirada.

## Archivos (todos del frontend)

- `src/services/api.js`
- `src/views/LoginView.jsx` (nuevo)
- `src/App.jsx`
- `src/hooks/useControlProduccion.js` (si gestiona el estado global, integrar la sesión ahí)

## Pasos

1. En `api.js`:
   - Función `login(username, password)` → `POST /auth/login`; guarda `accessToken` y `usuario` en memoria + `sessionStorage` (clave única, p. ej. `cp_session`).
   - `request()` añade `Authorization: Bearer <token>` si hay sesión.
   - Manejo de errores: parsear el body JSON del backend y lanzar `Error(message)` legible (ya no el texto crudo — el formato es `{ statusCode, message, ... }`).
   - Ante respuesta **401** en cualquier request: limpiar la sesión y volver al login (sin dejar la app en estado roto).
   - Función `logout()`: limpia memoria y `sessionStorage`.
2. `LoginView.jsx`: formulario username/contraseña, estilo consistente con las vistas existentes (mismas clases/componentes `Field`, `PageTitle`); muestra el mensaje de error del backend (p. ej. "Credenciales inválidas"); deshabilita el botón mientras envía.
3. `App.jsx`: si no hay sesión → renderiza solo `LoginView`; con sesión → la app actual + botón "Cerrar sesión" visible (puede ir en la barra de navegación existente).
4. Restaurar sesión desde `sessionStorage` al cargar la página.

## Criterio de aceptación

- [x] Flujo completo en navegador: login → operar vales/pagos/ventas normalmente → cerrar sesión → vuelve al login. *(verificado por UAT del usuario; ver registro)*
- [x] Con el token borrado o expirado, la siguiente acción devuelve al login limpiamente.
- [x] Credenciales malas muestran mensaje claro, sin romper la vista.
- [x] Commit (en el repo del frontend): `feat(auth): login screen, bearer token and session handling`.
