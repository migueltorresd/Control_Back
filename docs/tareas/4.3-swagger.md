# Tarea 4.3 — Documentación de API con Swagger

> Fase 4 — Operabilidad · 🟡 Ítem 12 del plan · Depende de: fase 2 · Reglas: [README.md](README.md)

## Contexto

No hay documentación de la API. Swagger facilita la entrega, el mantenimiento futuro y el desarrollo de la landing/storefront cuando llegue.

## Archivos

- `src/main.ts` (setup de Swagger)
- DTOs y controllers (decoradores)
- `src/config/env.validation.ts`, `.env.example` (flag de habilitación)

## Pasos

1. `pnpm add @nestjs/swagger`.
2. Setup en `main.ts`: `DocumentBuilder` con título, versión `1.0`, `addBearerAuth()`; UI en `/api/docs`.
3. **Condicionar por entorno**: habilitado solo si `NODE_ENV !== 'production'` o si `SWAGGER_ENABLED=true` (variable opcional en Joi, default false en producción) — una API interna no expone su documentación públicamente (OWASP A05).
4. Decorar los DTOs con `@ApiProperty` (descripciones en español, ejemplos reales: `V-0001`, `REF-001`) y los controllers con `@ApiTags`, `@ApiOperation`, `@ApiBearerAuth()`.
5. Los endpoints públicos (`/auth/login`, `/health`) marcados como tales en la doc.

## Criterio de aceptación

- [ ] `/api/docs` en desarrollo: permite hacer login, autorizar con el token (botón Authorize) y ejecutar `GET /api/v1/vales` desde la UI.
- [ ] Con `NODE_ENV=production` y sin flag, `/api/docs` → 404.
- [ ] Commit: `docs(api): swagger with bearer auth, disabled in production by default`.
