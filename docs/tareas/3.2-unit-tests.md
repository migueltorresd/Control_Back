## Cobertura mínima requerida

**`produccion.service.spec.ts` — máquina de estados completa (`updateEstado`):**
- `registrado → aprobado`: congela `montoPagado = pares × tarifa` del oficio correspondiente.
- `registrado → aprobado` sin tarifa definida para el oficio → `BadRequestException`.
- `aprobado → registrado`: vuelve el monto a 0.
- `aprobado → pagado` **sin** manager → `BadRequestException`; **con** manager → procede manteniendo el monto.
- `pagado → aprobado` sin manager → rechazado; con manager → procede.
- Transiciones inválidas (`registrado → pagado`, `pagado → registrado`) → `BadRequestException`.
- Registro de otro vale (`valeId` no coincide) → `NotFoundException`.
- Estado igual al actual → no-op que retorna el registro.

**Cupo (`registerProduccion`):**
- Registrar exactamente hasta el límite del vale → pasa.
- Un par por encima del límite → `BadRequestException`.
- Vale u operario inexistente → `NotFoundException`.

**`pagos.service.spec.ts`:**
- `pagar` con registro en estado ≠ `aprobado` → `BadRequestException`.
- `pagarLote` con inconsistencia vale/etapa → `BadRequestException`.
- `anularPagoPorRegistro` sin pago existente → `NotFoundException`.

**`deleteRegistro`:** registro `pagado` → `BadRequestException`.

**`vales.service` / `ventas.service`:** creación feliz + entidad referenciada inexistente → `NotFoundException`.
