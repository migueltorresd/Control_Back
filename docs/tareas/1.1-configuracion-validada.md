# Tarea 1.1 — Configuración única, tipada y validada (fail-fast)

> Fase 1 — Integridad de datos · 🟠 Ítem 8 del plan · Depende de: 0.1 · Reglas: [README.md](README.md)

## Contexto / problema

Hoy la configuración tiene tres defectos:

1. `src/app.module.ts:30-34` lee `process.env` con fallbacks silenciosos a credenciales débiles (`|| 'postgres'`). En producción, si falta una variable, se conecta con `postgres/postgres` sin avisar.
2. `src/config/database.config.ts:10` tiene un bug: `process.env.DATABASE_SYNCHRONIZE === 'true' || true` siempre evalúa a `true`. Además este config registrado con `registerAs('database')` **no se usa en ningún lado** — está duplicado y muerto.
3. Nada valida el entorno al arrancar: una variable mal escrita pasa desapercibida.

Patrón objetivo: **config fail-fast** — la app no arranca con configuración inválida (ver sección Patrones grupo B del plan maestro).

## Archivos

- `src/config/env.validation.ts` (nuevo)
- `src/config/database.config.ts` (rehacer o eliminar)
- `src/app.module.ts`
- `.env.example`

## Pasos

1. `pnpm add joi`.
2. Crear `src/config/env.validation.ts` con el esquema Joi:
   - `DATABASE_HOST`: string requerido.
   - `DATABASE_PORT`: número requerido.
   - `DATABASE_USERNAME`: string requerido.
   - `DATABASE_PASSWORD`: string requerido.
   - `DATABASE_DATABASE`: string requerido.
   - `PORT`: número, default 3001.
   - `NODE_ENV`: `development | production | test`, default `development`.
3. Pasarlo a `ConfigModule.forRoot({ isGlobal: true, validationSchema })`.
4. Cambiar `TypeOrmModule.forRoot(...)` por `TypeOrmModule.forRootAsync` que inyecte `ConfigService` y lea de ahí (un único punto de lectura de configuración). **Eliminar todos los fallbacks** `|| 'postgres'` etc.
5. **Eliminar** `src/config/database.config.ts` (está muerto: nadie lo consume y tiene el bug del `|| true`). La única fuente de config es `ConfigService` leído en el `forRootAsync`. El flag `synchronize` queda como está por ahora (`true`); se elimina en la tarea 1.2.
6. Mensaje de error de Joi claro: que diga qué variable falta.

## Criterio de aceptación

- [ ] Renombrar `.env` temporalmente → la app **no arranca** y el error indica la variable faltante. Restaurar `.env` → arranca normal.
- [ ] `grep -r "process.env" src/` solo aparece en la capa de config (y `main.ts` para `PORT` si se decide dejarlo, idealmente también vía ConfigService).
- [ ] Build y lint verdes. Commit: `feat(config): fail-fast env validation with joi`.
