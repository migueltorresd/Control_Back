# Tarea 0.1 — Línea base

> Fase 0 — Preparación · Depende de: nada · Reglas: [README.md](README.md) · Plan maestro: [../PLAN-MEJORAS-CALIDAD.md](../PLAN-MEJORAS-CALIDAD.md)

## Contexto

Antes de tocar nada, dejar el repo en un estado verificable y documentar las variables de entorno (hoy `.env` existe localmente pero no hay plantilla versionada — un desarrollador nuevo no sabe qué variables necesita).

## Pasos

1. Crear la rama de trabajo: `git checkout -b mejoras/fase-1`.
2. Verificar la línea base: `docker compose up -d`, `pnpm build` y `pnpm start:dev` funcionan; `GET http://localhost:3001/vales` responde.
3. Crear `.env.example` en la raíz del backend, versionado en git, con todas las variables actuales y valores de ejemplo (no reales):

```env
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_USERNAME=postgres
DATABASE_PASSWORD=cambiar_en_cada_entorno
DATABASE_DATABASE=control_produccion
PORT=3001
```

4. Confirmar que `.env` sigue en `.gitignore` (ya lo está) y que no quedó versionado: `git ls-files | findstr .env` solo debe mostrar `.env.example`.

## Criterio de aceptación

- [ ] `.env.example` versionado en git.
- [ ] La app arranca y `GET /vales` responde 200.
- [ ] Commit: `chore: add .env.example and baseline verification`.
